### [QA] QA-003: `ToeplitzAccumulator.init` public API has no docstring - constantine/math/matrix/toeplitz.nim:213-248

**Location:** constantine/math/matrix/toeplitz.nim:213-248
**Severity:** Low
**Confidence:** 0.9

**Diff Under Review:**
```diff
+proc init*[EC, ECaff, F](
+  ctx: var ToeplitzAccumulator[EC, ECaff, F],
+  frFftDesc: FrFFTDescriptor[F],
+  ecFftDesc: ECFFT_Descriptor[EC],
+  size: int,
+  L: int
+): ToeplitzStatus {.raises: [], meter.} =
+  # Free existing allocations (defensive: handles accidental double-init)
+  if not ctx.coeffs.isNil():
+    freeHeapAligned(ctx.coeffs)
   ...
```

**Issue: **Public API entry point missing parameter documentation**

`init*` is a public function that allocates `size * L` buffers for both coeffs and points (two heap allocations of `size*L` elements each, plus a scratch buffer of `max(size, L)`). This is an important API with memory implications that should be documented.

The object type `ToeplitzAccumulator*` itself has a brief `##` docstring ("Accumulator for Toeplitz matrix-vector multiplication with MSM") and good inline field comments, but the `init` procedure — which is the primary constructor — has no docstring explaining what `size` and `L` represent or the memory allocation behavior.

**Suggested Change:** Add a `##` docstring to `init*` explaining:
- What `size` is (the transform size, must be a power of 2)
- What `L` is (number of accumulation steps / phases)
- Memory allocation: ~`size * L * (sizeof(F) + sizeof(ECaff)) + max(size, L) * sizeof(F)` bytes allocated
- That it handles double-init defensively (already noted in inline comment)

### [QA] QA-004: `ToeplitzAccumulator.finish` docstring is insufficient - constantine/math/matrix/toeplitz.nim:272-276

**Location:** constantine/math/matrix/toeplitz.nim:272-276
**Severity:** Low
**Confidence:** 0.9

**Diff Under Review:**
```diff
+proc finish*[EC, ECaff, F](
+  ctx: var ToeplitzAccumulator[EC, ECaff, F],
+  output: var openArray[EC]
+): ToeplitzStatus {.raises: [], meter.} =
+  ## MSM per position, then IFFT
```

**Issue: **Public API docstring is too terse for a critical entry point**

The docstring "MSM per position, then IFFT" is only 5 words and doesn't explain:
- That this must be called after exactly `L` `accumulate` calls (the invariant `ctx.offset == ctx.L`)
- That the IFFT is performed in-place on the output buffer
- That the operation is variable-time (MSM is vartime)
- The preconditions and error conditions

The inline code comments are excellent (invariant about `scratchScalars` casting, MSM formula), but these aren't discoverable through Nim's doc generation tools.

**Suggested Change:** Expand the `##` docstring to include:
- Description: "Completes the accumulation by performing MSM at each position and an in-place IFFT"
- Preconditions: "Must be called after exactly `L` `accumulate` calls"
- Error: "Returns `Toeplitz_MismatchedSizes` if `offset != L` or output size doesn't match"

**Issue: **Comment slightly misleading after behavioral change**

The comment says "Runtime validation: prevent out-of-bounds indexing of agg_cols heap allocation." This is still accurate — the `doAssert` statements do prevent out-of-bounds at runtime (in debug builds). However, the function's return type changed from `bool` to `void`, and the caller was updated accordingly. The word "validation" here implies a checked operation that can fail gracefully, but `doAssert` is a hard failure (abort in release builds).

This is a minor concern since the intent (preventing OOB) is still correctly captured. The error messages are excellent and actionable.

**Suggested Change:** Consider rewording to "Invariant check: prevents out-of-bounds indexing of agg_cols heap allocation" to more accurately reflect that these are invariants, not graceful validations. Or leave as-is since the practical effect is the same.
