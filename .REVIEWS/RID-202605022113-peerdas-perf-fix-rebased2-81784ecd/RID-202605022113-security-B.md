This diff is a performance optimization pass focused on reducing allocations, switching to variable-time batch inversion where safe, and restructuring the FK20 Toeplitz multiplication to use an accumulator pattern with MSM. All changes were verified for security impact:

### `batchAffine_vartime` Usage Analysis (All Call Sites)

Each call site was verified to process only public data:

1. **`eth_verkle_ipa.nim:225`** — IPA proving: L/R are public proof points written to the proof object. Batch inversion timing depends only on z=0 detection (never true for valid IPA points).
2. **`eth_verkle_ipa.nim:260`** — IPA proving final round: `gg` are public generator-derived points; scalars `xinvbig` are challenge-derived (public).
3. **`eth_verkle_ipa.nim:649`** — IPA aggregation: `tmp` are public commitment sums.
4. **`kzg.nim:363`** — KZG batch verification: `commits_min_evals_jac` are public commitments minus public evaluations.
5. **`kzg_multiproofs.nim:415`** — FK20 proof output: `u` contains public proof points after IFFT.
6. **`kzg_parallel.nim:170`** — Parallel KZG verification: same data as `kzg.nim` path.
7. **`ec_scalar_mul_vartime.nim:365,427`** — Scalar mul precomp tables: points P, 2P, 3P… derived from public base point P. Timing leak of z-coordinates reveals nothing about the secret scalar (wNAF digit-dependent timing dominates).

### `computeAggRandScaledInterpoly` Error Handling Change

The function changed from `bool` return with `if ... return false` guards to `void` return with `doAssert` guards. In release builds (`-d:release`), `doAssert` is compiled away, removing the inner validation layer. However:

- The sole caller `kzg_coset_verify_batch` (line 700) performs identical runtime validation at lines 652-664 before calling.
- `computeAggRandScaledInterpoly` is not exported (no `*`), so no external callers exist.
- The comment on line 527 explicitly documents the purpose: "Runtime validation: prevent out-of-bounds indexing of agg_cols heap allocation" — the defense is now at the caller level.

This is a maintenance risk (future callers might omit validation) but not an exploitable vulnerability in the current codebase.

### `ToeplitzAccumulator` Memory Safety

- **`init`** (line 188): Defensive — frees existing allocations before re-initializing, preventing double-free on accidental double-init.
- **`=destroy`** (line 158): Nil-checks all three ptr fields before freeing, sets to nil after free.
- **`=copy {.error.}`**: Copy prohibited, preventing accidental double-free via copy-then-destroy.
- **`check`/`checkReturn`** templates: Structured error propagation with labeled break for cleanup. All allocations (`vExtFftAff`, `ifftResult`) are freed in the cleanup section after the HappyPath block.

### `bit_reversal_permutation` Aliasing Detection

New overload at `fft_common.nim:307` detects when `dst[0].addr == src[0].addr` and allocates a temporary buffer. Correctly allocated with `allocHeapArrayAligned`, copied back with `copyMem`, and freed with `freeHeapAligned`. No leak paths.

### `scratchScalars` Type Cast in `ToeplitzAccumulator.finish`

Line 284: `static: doAssert sizeof(F) == sizeof(F.getBigInt())` — compile-time check before `cast[ptr UncheckedArray[F.getBigInt()]]`. This is safe because:
- The size equality is guaranteed for all production field types (Fr[BLS12_381] is 32 bytes in both representations).
- The cast is only used to call `fromField()` which converts field elements to big integer scalars for MSM.
- The buffer is properly managed by the accumulator's `=destroy`.

### Positive Security Improvements

1. **`N ≤ 0` guards in all `batchAffine`/`batchAffine_vartime` variants**: Added early return for empty batches, preventing division-by-zero or out-of-bounds access in the Montgomery batch inversion algorithm.
2. **`checkCirculant` r=1 bounds fix**: `toeplitz.nim` now correctly checks `r+1 < k2` before accessing `circulant[r+1]`, fixing a latent out-of-bounds bug in the edge case where r=1.
3. **Eliminated `allocStackArray(SecretBool, N)`**: In `ec_twistededwards_batch_ops.nim`, the old code allocated a stack array of SecretBool for zero-tracking. The new code reuses `affs[i].y.mres.limbs[0]` as inline storage, eliminating a VLA that could compound stack usage under recursion.
4. **Proper `=copy {.error.}` on `ToeplitzAccumulator`**: Prevents a class of bugs where a copy would lead to double-free or use-after-free.

## Positive Changes

- **Defense-in-depth preserved despite `computeAggRandScaledInterpoly` simplification**: Caller-side validation in `kzg_coset_verify_batch` fully covers the removed inner checks.
